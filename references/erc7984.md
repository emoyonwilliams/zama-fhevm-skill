# ERC-7984 â€” Confidential Token Standard

## What is ERC-7984?

ERC-7984 is a confidential fungible token standard built on Zama's FHEVM. It is
similar to ERC-20 but designed from the ground up for privacy:

- All balances and transfer amounts are encrypted on-chain as `euint64` handles
- Not ERC-20 compliant by design â€” avoids ERC-20's transparency assumptions
- Implemented and audited by OpenZeppelin via `@openzeppelin/confidential-contracts`
- Powers `cUSDT` (the reward token for Zama Developer Program Season 2)

**Use ERC-7984 instead of a custom confidential token whenever possible.**
OpenZeppelin's implementation is audited, battle-tested, and handles ACL
correctly. Building your own risks subtle ACL bugs.

## Minimal ERC-7984 Token

```solidity
// SPDX-License-Identifier: BSD-3-Clause-Clear
pragma solidity ^0.8.24;

import { Ownable2Step, Ownable } from "@openzeppelin/contracts/access/Ownable2Step.sol";
import { FHE, externalEuint64, euint64 } from "@fhevm/solidity/lib/FHE.sol";
import { ZamaEthereumConfig } from "@fhevm/solidity/config/ZamaConfig.sol";
import { ERC7984 } from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";

contract MyConfidentialToken is ZamaEthereumConfig, ERC7984, Ownable2Step {
    constructor(
        address initialOwner,
        string memory name,
        string memory symbol,
        string memory contractURI
    ) ERC7984(name, symbol, contractURI) Ownable(initialOwner) {}

    /// @notice Mint tokens â€” amount encrypted from off-chain
    function mint(
        address to,
        externalEuint64 encryptedAmount,
        bytes memory inputProof
    ) external onlyOwner {
        _mint(to, FHE.fromExternal(encryptedAmount, inputProof));
    }

    /// @notice Burn tokens
    function burn(
        address from,
        externalEuint64 encryptedAmount,
        bytes memory inputProof
    ) external onlyOwner {
        _burn(from, FHE.fromExternal(encryptedAmount, inputProof));
    }
}
```

> Note: `ZamaEthereumConfig` and `ERC7984` both have constructors. Always call
> both with the correct parameters as shown above.

## Key ERC-7984 Interface Methods

```solidity
// Read encrypted balance (returns bytes32 handle, NOT a number)
euint64 handle = token.confidentialBalanceOf(userAddress);

// Transfer with inputProof (external encrypted amount)
token.confidentialTransfer(to, externalEuint64 amount, bytes inputProof);

// Transfer without inputProof (internal euint64 â€” used contract-to-contract)
token.confidentialTransfer(to, euint64 amount);

// Approve operator for any amount (no amount specified â€” operator pattern)
token.setOperator(spender, untilTimestamp);
bool approved = token.isOperator(holder, spender);

// Confidential total supply
euint64 totalSupply = token.confidentialTotalSupply();
```

## ERC-7984 Transfer Functions (8 variants)

ERC-7984 exposes 8 transfer permutations combining:
- `transfer` (from sender) vs `transferFrom` (from specified address)
- With `inputProof` (user-supplied encrypted amount) vs without (internal handle)
- With vs without operator approval check

```solidity
// Most common â€” user transfers their own tokens
confidentialTransfer(address to, externalEuint64 amount, bytes inputProof)

// Contract-to-contract (no proof needed â€” amount is already an internal handle)
confidentialTransfer(address to, euint64 amount)

// Spend from another address (requires operator approval)
confidentialTransferFrom(address from, address to, externalEuint64 amount, bytes inputProof)
confidentialTransferFrom(address from, address to, euint64 amount)
```

## Wrapping ERC-20 â†’ ERC-7984

Convert a plain ERC-20 into a confidential token using `ERC7984ERC20Wrapper`:

```solidity
import { ERC7984ERC20Wrapper } from
  "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984ERC20Wrapper.sol";
import { IERC20 } from "@openzeppelin/contracts/interfaces/IERC20.sol";

contract WrappedConfidentialUSDC is ZamaEthereumConfig, ERC7984ERC20Wrapper {
    constructor(IERC20 underlyingToken)
        ERC7984ERC20Wrapper(underlyingToken, "Confidential USDC", "cUSDC", "") {}
}
```

The wrapper allows free conversion in both directions at a 1:1 rate:
- ERC-20 â†’ ERC-7984: amount is visible going in, then becomes encrypted
- ERC-7984 â†’ ERC-20: requires explicit decryption step (amount becomes public)

## Swapping ERC-7984 â†’ ERC-20

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.27;

import { FHE, externalEuint64, euint64 } from "@fhevm/solidity/lib/FHE.sol";
import { IERC20 } from "@openzeppelin/contracts/interfaces/IERC20.sol";
import { SafeERC20 } from "@openzeppelin/contracts/token/ERC20/utils/SafeERC20.sol";
import { IERC7984 } from "@openzeppelin/confidential-contracts/interfaces/IERC7984.sol";

contract SwapConfidentialToERC20 {
    using SafeERC20 for IERC20;

    IERC7984 private _fromToken;
    IERC20   private _toToken;

    mapping(euint64 => address) private _receivers;

    constructor(IERC7984 fromToken, IERC20 toToken) {
        _fromToken = fromToken;
        _toToken   = toToken;
    }

    function swap(externalEuint64 encryptedInput, bytes memory inputProof) external {
        euint64 amount = FHE.fromExternal(encryptedInput, inputProof);
        _receivers[amount] = msg.sender;
        // Transfer confidential token from user to this contract
        _fromToken.confidentialTransferFrom(msg.sender, address(this), amount);
        // Decryption happens async via Gateway callback
    }

    // Called by Gateway after decryption
    function finalizeSwap(euint64 encryptedAmount, uint64 clearAmount) external {
        address receiver = _receivers[encryptedAmount];
        _toToken.safeTransfer(receiver, clearAmount);
    }
}
```

## Available ERC-7984 Extensions

| Extension | Purpose |
|---|---|
| `ERC7984ERC20Wrapper` | Wrap/unwrap plain ERC-20 â†” confidential ERC-7984 |
| `ERC7984Freezable` | Freeze/unfreeze accounts (compliance) |
| `ERC7984ObserverAccess` | Grant observer view access to balances |
| `ERC7984Restricted` | Blocklist/allowlist transfer restrictions |
| `ERC7984Rwa` | Real-world asset compliance + force transfer |
| `ERC7984Omnibus` | Encrypted sub-account transfers (exchange use case) |

Import pattern:
```solidity
import { ERC7984Freezable } from
  "@openzeppelin/confidential-contracts/token/ERC7984/extensions/ERC7984Freezable.sol";
```

## ACL in ERC-7984 â€” What the Base Contract Handles

The `ERC7984` base contract automatically manages ACL in `_update()`:
- Calls `FHE.allowThis()` on balance handles
- Calls `FHE.allow(handle, recipient)` for the receiver
- Calls `FHE.allow(handle, sender)` for the sender

**You do NOT need to call `FHE.allowThis` or `FHE.allow` manually when using
`_mint`, `_burn`, or `_transfer` from the base contract.** The base handles it.
You only need manual ACL calls if you store additional encrypted state outside
the standard balance/allowance mappings.

## Testing ERC-7984

```ts
import { ERC7984Token, ERC7984Token__factory } from "../types";
import { FhevmType } from "@fhevm/hardhat-plugin";
import { ethers, fhevm } from "hardhat";

describe("ERC7984Token", function () {
  it("mint and check confidential balance", async function () {
    const [deployer, alice] = await ethers.getSigners();

    const factory = await ethers.getContractFactory("MyConfidentialToken");
    const token = await factory.deploy(deployer.address, "MyToken", "MTK", "");
    const tokenAddress = await token.getAddress();

    // Encrypt mint amount
    const enc = await fhevm
      .createEncryptedInput(tokenAddress, deployer.address)
      .add64(1_000_000n)
      .encrypt();

    await token.connect(deployer).mint(alice.address, enc.handles[0], enc.inputProof);

    // Decrypt Alice's balance
    const handle = await token.confidentialBalanceOf(alice.address);
    const balance = await fhevm.userDecryptEuint(
      FhevmType.euint64, handle, tokenAddress, alice
    );
    expect(balance).to.eq(1_000_000n);
  });
});
```
